    Assessments grain split — PR 3 implementation plan

    For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

    Goal: Drop canonical_title, canonical_administered_at, canonical_grade_level_id, administered_date, and grade_level from int_assessments__assessments_members, and migrate consumers to join int_assessments__assessments_canonical for those values.

    Architecture: PR 1 added int_assessments__assessments_canonical (canonical-grain). PR 2 renamed the member model. PR 3 finishes the grain split by removing the denormalized canonical_* pass-through columns from _members; consumers that need canonical attrs now traverse the FK chain via canonical_assessment_id → assessments_canonical. Hash-stable for all 4 mart surrogate keys that previously hashed cast(canonical_administered_at as date) / administered_date from members.

    Tech stack: dbt 1.11 (BigQuery adapter), dbt_utils package, sqlfluff/trunk-fmt.

    Spec: docs/superpowers/specs/2026-05-27-assessments-grain-split-design.md

    Tracking issue: #3800 (closed by this PR).

    Worktree: .worktrees/cbini/refactor/claude-assessments-grain-split-pr3/. Every git command uses git -C <worktree>; every dbt command uses --project-dir <worktree>/src/dbt/kipptaf.
